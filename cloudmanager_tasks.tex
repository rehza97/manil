\section*{Objectif}
Livrer un MVP en 90 jours ouvrés avec: authentification complète (2FA, RBAC, audit), ticketing complet, système de notifications, gestion clients (KYC minimal), catalogue produits, commandes, facturation (devis/factures PDF), reporting de base (tableaux/CSV), et documentation.

  \item \textbf{Wassim} (Lead Frontend/Business): UI/UX React, Customer Manager, Product Catalogue, Order Manager, Invoice, Reporting, intégration API, maquettes et validations métier.
\end{itemize}

\section*{Détail des fonctionnalités par développeur}
\subsection*{Manil}
\begin{itemize}
  \item \textbf{Authentification, Sécurité et RBAC}
  \begin{itemize}
    \item Inscription/connexion email+mot de passe (hash Argon2), réinitialisation par e\-mail.
    \item 2FA TOTP (QR Code, codes de secours), option SMS (abstraction fournisseur).
    \item Sessions et appareils: journal, invalidation, durée d'expiration configurable.
    \item RBAC: rôles (client, corporate, admin) + permissions par action (lecture/écriture/export, transfert ticket, gestion modèles, etc.).
    \item Audit trail: connexions, changements de rôles, paramètres, opérations sensibles.
    \item Sécurité API: JWT access/refresh, CORS, rate limiting, validation Pydantic, sanitisation PJ.
  \end{itemize}
  \item \textbf{Ticket Manager (backend)}
  \begin{itemize}
    \item Modèle de données: tickets, messages, pièces jointes, tags, états.
    \item Cycle complet d'états (ouvert, répondu, attente, on\-hold, en cours, résolu, fermé) + re\-ouverture.
    \item Assignation groupes/agents, transfert, watchers, notes privées (visibilité admin).
    \item Canned replies/modèles, variables (client, service, ticket), snippets.
    \item Mail\-to\-ticket: récupération IMAP/webhook, association par Message\-ID, antispam simple.
    \item KPI de base: premier délai de réponse, délai de résolution, files par priorité/groupe.
    \item Exports: PDF d'un ticket (optionnel), CSV des listes.
  \end{itemize}
  \item \textbf{Settings et Notifications}
  \begin{itemize}
    \item Paramètres: catégories tickets, groupes support, fermeture auto après X jours, gabarits notification.
    \item Groupes de notifications: ciblage client/catégorie/général, envoi asynchrone.
    \item Moteur de templates e\-mail (Jinja2): variables, prévisualisation, historique d'envoi, rebonds de base.
  \end{itemize}
  \item \textbf{Infrastructure}
  \begin{itemize}
    \item Schéma PostgreSQL, migrations Alembic, indexations clés.
    \item Organisation du dépôt (monorepo front/back), scripts de démarrage, configuration .env.
    \item Packaging et déploiement simple (Docker Compose ou service systemd), sauvegardes DB.
  \end{itemize}
\end{itemize}

\subsection*{Wassim}
\begin{itemize}
  \item \textbf{Frontend (React)}
  \begin{itemize}
    \item Architecture UI: routing protégé, layouts par rôle, theming, composants réutilisables (Shadcn/UI).
    \item Gestion d'état (React Query/Zustand), formulaires typés (React Hook Form + Zod/Pydantic types).
    \item Upload sécurisé (affichage aperçu), toasts, pagination, filtres, recherche.
  \end{itemize}
  \item \textbf{Customer Manager}
  \begin{itemize}
    \item Profils clients, multi\-utilisateurs par client, validation KYC minimal (upload docs, statut, motif).
    \item Activation 2FA côté client (TOTP), affichage activités de sécurité et sessions.
    \item Fiche client (notes internes, documents, historiques tickets/commandes/factures).
  \end{itemize}
  \item \textbf{Product Catalogue}
  \begin{itemize}
    \item Pages publiques catalogue, catégories, fiche produit/service.
    \item Création compte depuis le catalogue; demandes de devis/services (workflow simple).
    \item Backoffice simple: CRUD produits, catégories, statut visibilité.
  \end{itemize}
  \item \textbf{Order Manager}
  \begin{itemize}
    \item Workflow demande $\to$ validation $\to$ en cours $\to$ livrée/annulée, commentaires, notifications.
    \item Liens contextuels vers client, tickets associés, devis/factures.
  \end{itemize}
  \item \textbf{Invoice et Reporting}
  \begin{itemize}
    \item Génération Devis/Factures PDF (modèle, numérotation), envoi e\-mail.
    \item Vues reporting: tableaux filtrables (tickets, clients, commandes), export CSV.
  \end{itemize}
  \item \textbf{Qualité et UX}
  \begin{itemize}
    \item Maquettes rapides, accessibilité de base, responsive.
    \item Guide utilisateur (captures, procédures), onboarding minimal.
  \end{itemize}
\end{itemize}
